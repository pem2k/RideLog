### Grading Rubric

| Criteria | Points |
| :--- | :--- |
| **Design Document**<br>Includes at least: Project description, User Personas, User stories (use cases with a story), and Design mockups. | 80 |
| **Requirements Met**<br>Does the app accomplish all the requirements approved in #project? | 15 |
| **Usability**<br>Is the app usable? Does it include instructions on how to be used? | 5 |
| **Usefulness**<br>Is the app actually useful? Would a final user use it? | 5 |
| **Linting**<br>The code includes an ESLint config file and it doesn't throw any errors. | 5 |
| **Code Organization**<br>Is the code properly organized? Each React component in its own file. Database files separate, CSS into folders, etc. | 5 |
| **Component Count**<br>Does it implement at least 3 React Components (using hooks)? | 15 |
| **Component Files**<br>Each React Component is saved into its own file. | 15 |
| **Deployment**<br>Is the project deployed on a public server? Does it work there? | 10 |
| **Database Operations**<br>Does it use at least 2 Mongo Collections, supporting CRUD operations on them? Each student implements full CRUD in at least one of them. | 15 |
| **Database Seeding**<br>Is the database populated with at least 1k synthetic records? | 10 |
| **Backend Stack**<br>Does the app use Node + Express? | 5 |
| **Code Formatting**<br>All the code is formatted using Prettier. | 5 |
| **Semantic HTML**<br>Doesn't use non-standard HTML tags for accomplishing tasks that should be done with standard components (e.g., Buttons implemented with divs or spans). | 5 |
| **CSS Organization**<br>Is the CSS organized by components? The components that require styling should have their own CSS file with the same name of the component and loaded in the component's JS file. | 5 |
| **Documentation**<br>Clear and descriptive README including at least: Author, Class Link, Project Objective, Screenshot, and Instructions to build. | 5 |
| **Security**<br>It doesn't expose any secret credentials such as the Mongo USER and Password. | 5 |
| **Dependencies**<br>Does it include a `package.json` file for the backend and one for the frontend, listing all the dependencies for each one? | 5 |
| **License**<br>Does it use an MIT license? | 5 |
| **Clean Codebase**<br>The project doesn't include leftover code that isn't used (e.g., `routes/users.js` or the default React favicon). | 5 |
| **Form Submission**<br>Is the Google Form submission correct (thumbnail displays correctly, and links work as well)? | 5 |
| **Video Demonstration**<br>Does it include a short public and narrated video demonstrating the application (each student present on it)? | 10 |
| **Timeliness**<br>The code was frozen on time (24 hours before class). Video, slides, and deployment were completed before class. | 5 |
| **Type Checking**<br>Defines PropTypes for every React Collection. | 5 |
| **Library Restrictions**<br>Doesn't use axios, Mongoose, CORS, or any other prohibited library. *(Note: Document text lists this as a -20 point penalty if violated, but the scoring grid assigns 20 points for compliance).* | 20 |
| **Authentication**<br>Implements authentication using Passport as explained in the videos. | 20 |
| **Total Points** | **290** |
